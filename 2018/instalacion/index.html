Instalación y configuración de hugo

Hugo es un generador de páginas web estáticas, está escrito en lenguaje de programación Go, licencia APL 2.0, con un generador de plantillas Go Templates.

Instalación

Para instalar hugo ejecutamos la siguiente instrucción:

$ sudo apt-get install hugo

Creación sitio web

$ mkdir sitioweb
$ hugo new site sitioweb

El ultimo comando nos creara toda la estructura del sitio web, dentro de la carpeta sitioweb.

$ ls
archetypes  config.toml  content  data  layouts  static  themes

Creación primer post

Agregamos un post para ver el sitio web que acabamos de crear, ejecutamos lo siguiente:

$ hugo new post/instalacion.md

Ahora editamos el fichero /content/post/instalacion.md, por defecto viene el siguiente contenido:

+++
title = "instalacion"
draft = true
date = "2018-09-26T17:06:21Z"

+++

El contenido dentro +++ es la configuración TOML. Agregamos texto despues de +++ en formato Markdown, Online Markdown Editor

+++
title = "instalacion"
draft = true
date = "2018-09-26T17:06:21Z"

+++

### Instalación
Para instalar hugo ejecutamos la siguiente instrucción:
```sh
$ sudo apt-get install hugo
```

Agregar un tema

Hugo tiene una muy robusta y versátil librería de temas pues usa la librería html/template de Go. Puedes descargar una plantilla pinchando aqui. Para agregar un tema ejecutamos lo siguiente en nuestro directorio themes:

$ git clone https://github.com/Tazeg/hugo-blog-jeffprod.git themes/jeffprod

Fichero config.toml

En este fichero esta la configuración principal, podemos definir el título de todos los sitios web, idioma, urls, etc.

title = "Implantacion aplicaciones web"
baseurl = ""
languageCode = "en-us"
theme = "jeffprod"
[permalinks]
    post = "/:year/:filename"

[taxonomies]
    tag = "tags"
    archive = "archives"

Desplegar hugo

Para ejecutarlo en modo desarrollo ejecutamos lo siguiente:

$ hugo server -D

Accedemos a http://localhost:1313 Si tenemos hugo en una maquina virtual y no podemos ver nuestra pagina le decimos con el siguiente comando que escuche en la 0.0.0.0

$ hugo server -D --bind=0.0.0.0

Para llevarlo a producción y tener el árbol de ficheros ejecutamos lo siguiente:

$ hugo -D

Vemos como nos crea una carpeta public.

$ ls
archetypes  config.toml  content  data  layouts  *public*  static  themes

Ejecutamos el siguiente comando para comprobar que nuestra pagina esta funcionando:

hugo server -D --bind=0.0.0.0 -b http://172.22.0.108
